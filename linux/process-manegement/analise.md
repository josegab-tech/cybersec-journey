# Day 3 - Processes in Linux

## Objective
Understand how processes work and how to manage them.

## Commands Used
ps
ps aux
ps aux | grep sleep
sleep 200 &
kill PID
top

## Test Performed
Created a process using sleep command and monitored it.

## Results
The process was successfully identified and terminated.

## Learning
Processes represent running programs.
It is possible to monitor and control them.

## Notes
Monitoring processes is important to detect abnormal behavior.
